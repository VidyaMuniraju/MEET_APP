Introduction

  2. Show/hide event details (implemented in Exercises 4.2 and 4.3)

With this Exercise, you’ll implement the last remaining feature, “it must display a chart showing the number of upcoming events by city,” using a technique called data visualization. Not only will this look pretty darn cool in your app, it will also help users engage more easily with the information displayed in your app. This kind of data engagement has become a bit of a hot topic in the field, lately, making data visualization a skill you’ll want to (and will!) add to your developer toolbox.

You’ll do so by diving into some React-specific data visualization libraries, using one of them to add a chart to your Meet application. By the end of this Exercise, you’ll be able to use this library (along with many others) to add any kind of chart or graph your heart desires to your future React projects. Sound like fun? Then, let’s get started!

Charting Libraries

If you’ll remember from Exercise 3.1: Intro to Frameworks & Libraries, a library is a package that can be integrated into a project or app in order to speed up the development process. Libraries come with predefined components you can use so that you don’t have to implement them from scratch. The Bootstrap library, for instance, provides many common CSS elements such as labels, buttons, sliders, and more for styling websites. Even React calls itself a library (though it certainly acts more like a full-fledged framework)!

In this Exercise, you’ll be looking specifically into charting libraries used for data visualization. “Charting,” here, refers to the act of turning data into a chart of some sort, whether that be a line graph, pie chart, waterfall chart, or otherwise.

D3 vs. React-Specific Charting Libraries

D3 is one of the most popular JavaScript libraries for manipulating documents based on data—it can even be used for more advanced functionality such as animation and data analysis. It uses web standards (HTML, SVG, and CSS) to visualize almost anything you could possibly want straight within the browser. Its one drawback, however, is that you need to draw these things manually. It won’t create charts for you based on your data—you have to draw them yourself.

This is where the React-specific charting libraries you’ll be looking at in this Exercise come into play. While all of these charting libraries are built on the D3 engine, so to speak, they go above and beyond the core functionality of D3 itself. Rather than having to draw your own charts manually, you can use predefined components to create what are essentially “out-of-the-box visualizations,” cutting down on development time (and acting as a much-needed savior for developers who may not be artistically inclined).

In D3, for example, if you wanted to draw a bar chart, you’d need to draw a bunch of rectangles, position them at the right coordinates, and paste some textual elements on top, which may or may not end up looking like the masterpieces you used to draw in Microsoft Paint. With a React-specific library, however, all you need to do is provide a set of data, choose a chart type, and let the library take care of the rest—a win for everyone!

The question, then, is which React-specific library is most suited to the job? There are quite a few of them out there, and they all have their own advantages and disadvantages in certain situations. Let’s go over a few criteria you’ll be able to use to make an informed decision.

  • What kind of chart do you want to visualize? Basic charts (line charts, bar charts, pie charts, etc.) or more advanced charts (radar charts, scatter charts, radial bar charts, etc.)? You already know you want to create a chart that displays the number of upcoming events by city. For this, you’ll need a two-dimensional chart—one dimension for the date and one dimension for the number of events. A two-dimensional chart could take a number of different forms (line chart, bar chart, scatter chart, etc.), so any chart library that supports one of these options would be a possible candidate.
  • Will the chart be displayed on desktop, mobile devices, or both? You want your app to work on both desktop and mobile devices, so you need a chart that’s responsive—no matter which device it’s displayed on, it needs to look good.
  • Which JavaScript framework did you use? You used React to create your Meet application, so you’ll need to go with a React-specific library.

From the above analysis, one can derive that a successful library candidate would need to 1) support two-dimensional charts, 2) be responsive, and 3) work well with React. Truth be told, there are any number of React-specific libraries that satisfy these conditions. In cases like these, it pays to research which libraries are the most popular, as a greater degree of popularity means more documentation and support if you end up running into trouble.

The top three React-specific charting libraries are:

  • Recharts: A lightweight charting library that’s both powerful and reliable. It supports a wide variety of charts (area charts, bar charts, line charts, composed charts, pie charts, radar charts, radial bar charts, scatter charts, funnel charts, treemaps, and more), as well as responsive charts.
  • Victory: A charting library for modular charting and data visualization, it renders components into responsive containers by default and supports a wide variety of charts.
  • React-Vis: A simple and easy-to-use graph library created by Uber. It supports responsive charts and, like the others, a wide variety of chart types.

Each of the libraries above are well-documented, open source, free to use, and easy to customize. They also each have a high number of GitHub stars—a good metric for measuring community value. This Exercise will focus on Recharts, as it has the highest number of GitHub stars, making it the most popular React data visualization library on the market today—a claim that can be backed up by looking at the number of downloads in the last six months:

Figure 1. Source: NPM Trends

Choosing a Chart

Before you can add a chart to your project, you first need to choose a type of chart. As mentioned earlier, a two-dimensional chart can be a line chart, bar chart, area chart, or more—but as all of those are pretty standard types of chart, why not try something a little fancier? For your Meet app, you’ll create what’s called a scatterplot (referred to in Recharts as a “ScatterChart”), which is a very versatile chart that can display up to four dimensions (x-axis, y-axis, point size, and point color).

Head over to Rechart’s “SimpleScatterChart” page to get a better idea of what you’ll be building:

Figure 5

In that same page, you’ll find some sample code for drawing a scatterplot of your own. You can see it below:

import React, { PureComponent } from 'react';
import {
  ScatterChart, Scatter, XAxis, YAxis, CartesianGrid, Tooltip
} from 'recharts';

const data = [
  { x: 100, y: 200, z: 200 },
  { x: 120, y: 100, z: 260 },
  { x: 170, y: 300, z: 400 },
  { x: 140, y: 250, z: 280 },
  { x: 150, y: 400, z: 500 },
  { x: 110, y: 280, z: 200 },
];

export default class Example extends PureComponent {
  static jsfiddleUrl = 'https://jsfiddle.net/alidingling/uLysj0u2/';

  render() {
    return (
      <ScatterChart
        width={400}
        height={400}
        margin={{
          top: 20, right: 20, bottom: 20, left: 20,
        }}
      >
        <CartesianGrid />
        <XAxis type="number" dataKey="x" name="stature" unit="cm" />
        <YAxis type="number" dataKey="y" name="weight" unit="kg" />
        <Tooltip cursor={{ strokeDasharray: '3 3' }} />
        <Scatter name="A school" data={data} fill="#8884d8" />
      </ScatterChart>
    );
  }
}

The first line of code imports React and PureComponent from the React library. This is necessary to render the chart, as you won’t be able to do much without a component. As this component won’t do anything besides render(), you can use a PureComponent instead of a normal Component. Inside of this component is a static property: static jsfiddleUrl = 'https://jsfiddle.net/alidingling/uLysj0u2/';. The only purpose of this is to link to the scatterplot chart code on jsfiddle.

After this comes a few more imports, this time from Recharts. ScatterChart is imported as the container; Scatter is imported to draw the points; XAxis and YAxis are imported for the horizontal and vertical axes respectively; CartesianGrid is imported to draw the rectangular coordinate system; and Tooltip is imported to reveal information about the chart on hover:

import {
  ScatterChart, Scatter, XAxis, YAxis, CartesianGrid, Tooltip
} from 'recharts';

After this is a set of mock data, which represents a list of points to be drawn on the chart:

const data = [
  { x: 100, y: 200, z: 200 },
  { x: 120, y: 100, z: 260 },
  { x: 170, y: 300, z: 400 },
  { x: 140, y: 250, z: 280 },
  { x: 150, y: 400, z: 500 },
  { x: 110, y: 280, z: 200 },
];

Each point is an object with three properties: x represents the x coordinate, y represents the y coordinate, and z represents the z coordinate. Even though z is normally used to create a third-dimension within charts and graphs (and you only need to create a two-dimensional graph), within a scatterplot, it also controls the size of a point. For this reason, go ahead and leave it for now, as you may want to play with that property later to create some neat effects.

Next up is the code that actually renders the chart:

  render() {
    return (
      <ScatterChart
        width={400}
        height={400}
        margin={{
          top: 20, right: 20, bottom: 20, left: 20,
        }}
      >
        <CartesianGrid />
        <XAxis type="number" dataKey="x" name="stature" unit="cm" />
        <YAxis type="number" dataKey="y" name="weight" unit="kg" />
        <Tooltip cursor={{ strokeDasharray: '3 3' }} />
        <Scatter name="A school" data={data} fill="#8884d8" />
      </ScatterChart>
    );
  }

Each component in Recharts is a React component, meaning that you can use them the same as you would any normal React component. The key here, then, is knowing which properties you can customize. With scatterplots, for example, you can customize the width, height, and margin–but how? This is where the Recharts documentation comes into play.

Each of the components listed can be further researched on their own individual pages for even more information; for instance, to look up how to customize the XAxis component, simply head on over to the XAxis API page. You’ll see that it uses the type property to specify the format of the data to be rendered. The code type="number" means you’ll provide it with data in the form of numbers, while the code dataKey="x" specifies that XAxis will take the value of the x attribute in the mock data. The YAxis component follows a similar format.

Both of them also take name and unit properties to display more meaningful information about the data. Take a look at the two charts below, the left of which doesn’t have name and unit properties and the right of which does. See how much more context the numbers and points on the chart have now?

Figure 11

There are a few other components you can add without specifying any properties, like the CartesianGrid. This doesn’t mean that the component doesn’t have any properties—simply that they’re optional (e.g., a property designating solid or dashed lines). Wondering what CartesianGrid does? Check out the images below, which denote a chart without and with a cartesian grid respectively:

Figure 12

The cartesian grid option plots the grid lines along the x and y axis in dashed or solid lines. The grid lines help make it easy for the user to follow the points along the axes and easily understand the values along the two dimensions.

The last component listed is also the most important—Scatter. Without this component, your chart won’t have any points, making it pretty darn useless. You’ll see that the example code uses three different properties: name, data, and fill:

<Scatter name="A school" data={data} fill="#8884d8" />

The name is what those dots will represent. With a chart that only displays data for one thing, this isn’t as important, but with a chart that compares data from different sets, it’s essential, as otherwise, there’s no way to know which dots are which. Likewise, the fill property can be used to differentiate data from different sets, as it’s what sets the fill color for your points.

Then comes a special property—data. This property is what passes the specified array of data to the chart. Here, the line data={data} means that the previously defined mock data is what’s being passed into the chart. If you look at the chart itself, you’ll see six points, the same as the six objects in the mock data array.

Data for the data property comes in the form of an array of objects. Each object can have as many attributes as you want (you can choose which attributes to use via the dataKey property) so long as they come in the format of a string or number—the only two value formats accepted by Recharts.

Take a look at the set of data below, which will plot two different points on the chart:

const data = [
  {
    name: '18-24', uv: 31.47, pv: 2400, fill: '#8884d8',
  },
  {
    name: '25-29', uv: 26.69, pv: 4567, fill: '#83a6ed',
  },
];

Now, take a look at an example of invalid data (the value of info is an object rather than a string or number):

const data = [
  {
    name: 'Page A', info: { uv: 4000, pv: 2400, amt: 2400 },
  },
  {
    name: 'Page B', info: { uv: 3000, pv: 1398, amt: 2210 },
  },
];

No matter what, you’re going to need some data—data from the Google Calendar API. Let’s get to work preparing just that!

Preparing Your Data

You already know this feature of your app “must display a chart showing the number of upcoming events by city.” This was stated in the project brief. In order to accomplish this, you need to know how many events will take place in each city for all upcoming events.

Let’s look at how you should format your events data to display in a ScatterChart. You’ll need two values—the XAxis component for the location names (though they’ll need to be shortened to fit on the plot), and YAxis component for the number of events. This means your data will need to look something like this:

const data = [
  { "city": "Dubai", "number": 2 },
  { "city": "Toronto,", "number": 2 },
  { "city": "Santiago,", "number": 3 },
  { "city": "Tokyo,", "number": 2 }
]

Of course, this is just a made-up example—you’ll need the real number of events in each city from the Google Calendar API to actually feed into the plot. Fortunately, the API makes this fairly simple. Open up your “mock-data.js” file and look at the location attribute included with each event. This attribute represents the city or location of the event:

Figure 13

TIP!
It may have been a couple of days since you first made the mock data for your event. You might go ahead and update this data now to match the current date. Keeping your mock data updated during development helps greatly when working with date-specific data, especially in cases like this, where you specifically need to load data for events in the future. For a refresher on where and how to get mock data for your events, reread the section in Exercise 4.4: Integration Testing. You’ll need to use the Google Calendar API console again!

In order to get the total number of events happening in each city, you need to count how many events each city has. Let’s write a function to do just that! In the App component (your “App.js” file), add the following function:

  getData = () => {
    const {locations, events} = this.state;
    const data = locations.map((location)=>{
      const number = events.filter((event) => event.location === location).length
      const city = location.split(' ').shift()
      return {city, number};
    })
    return data;
  };

This function takes advantage of the work you’ve already done and uses the locations and events saved in your state from the Google Calendar API. You then map the locations and filter the events by each location to get the length of the resulting array. To get the city, you need to shorten the location and remove any unnecessary information so you only have a city name.

To do this, you split the location at every space, which will return an array. Then, you simply use the shift() array function to get the first element in the array, which is the name of the city. The data being returned will be an array with an object of {city, number}.

Great! Your data is ready to go—all that’s left to do is add it to the chart.

Creating and Customizing Your Chart

When working with Recharts, the quickest way to get started with a chart is to simply copy-paste the example code given into your own project, then customize it to meet your needs. Let’s do this now.

Adding the Sample Chart

You already know you want to draw your chart on top of the event list in the UI of your App component (so that it will be displayed along with the list of events). Go ahead and copy-paste the code in the ScatterChart example into your “App.js” file above <EventList> (inside the render function). It should look like this when you’re done:

render() {
    const { locations, numberOfEvents } = this.state;
    return (
      <div className="App">
        <h1>Meet App</h1>
        <h4>Choose your nearest city</h4>
        <CitySearch updateEvents={this.updateEvents} locations={locations} />
        <NumberOfEvents
          updateEvents={this.updateEvents}
          numberOfEvents={numberOfEvents}
        />       
         <h4>Events in each city</h4>

         <ScatterChart
          width={400}
          height={400}
          margin={{
            top: 20, right: 20, bottom: 20, left: 20,
          }}
        >
          <CartesianGrid />
          <XAxis type="number" dataKey="x" name="stature" unit="cm" />
          <YAxis type="number" dataKey="y" name="weight" unit="kg" />
          <Tooltip cursor={{ strokeDasharray: '3 3' }} />
          <Scatter name="A school" data={data} fill="#8884d8" />
        </ScatterChart>

      </div>
    );
  }

Don’t forget to import these components at the top of the file, as well:

import {
  ScatterChart, Scatter, XAxis, YAxis, CartesianGrid, Tooltip
} from 'recharts';
Customizing the Sample Chart

You now have a shiny new chart in your app, but it won’t do you much good, yet, as it’s still only displaying the example data. Let’s customize it a bit to make it more suited to the data you’ll be adding in the last step.

Start by looking at the line:

<Scatter name="A school" data={data} fill="#8884d8" />

This is what defines your scatterplot. Replace the value of data with the new getData function you wrote above, like this: <Scatter data={this.getData()} fill="#8884d8" />. Also, as you’re only displaying one set of data on this chart, you don’t need a name, so you can delete that property for now.

Next up are <XAxis> and <YAxis>. These need to be updated to use the correct attributes from your data set. Change the dataKey of <XAxis> to city instead of x, as the attribute’s name is city in the object. Similarly, change type to category.

The x-axis will represent cities, which are expressed as strings in your data. Strings in Recharts take the type category. Lastly, you can delete the unit attribute, as that’s no longer necessary. You should end up with an <XAxis> line that looks something like this:

<XAxis type="category" dataKey="city" name="city" />

For <YAxis>, you’ll want to set the dataKey to number, as this axis will be what represents the number of events in that city. As the type attribute is already set to number, you don’t need to change it. Finally, go ahead and set the name property to something descriptive, like number of events. This way, if you include a tooltip, users will be able to see more information on what the chart represents. Just like <XAxis>, you can also delete the unit attribute. In the end, you should end up with something like this:

<YAxis type="number" dataKey="number" name="number of events" />

TIP!
If you see decimal numbers showing on the y-axis (e.g., 0.5, 1.5, 2.5, etc.), this is wrong. Your number should be an integer—not a float. Add this property to <YAxis> to eliminate the decimals:

allowDecimals={false}

Once you’ve made all of these changes, deploy your app locally to see how it looks using npm run start in your terminal. Open up your app in your browser (“http://localhost:3000”), and you should see your new chart!

Figure 14

Amazing! But wait! Why are only five cities showing? This is because the chart isn’t wide enough (it’s currently only 400px), so Recharts has automatically hid the days that don’t fit. This isn’t what you want, though—you want to see the whole chart!

To fix this, update the width of your chart to 800px (width={800}), then re-deploy it and refresh your browser. Are you seeing more cities?

Figure 15

Great! It looks pretty good. You may notice, however, that if you change the width of your browser (or if you load your app on a mobile device), the chart is cut off. This isn’t exactly ideal, as you want your app to look good on any screen size. The chart currently isn’t responsive. Let’s fix that!

Making Your Chart Responsive

As mentioned above, the only parent component your scatterplot can take is ResponsiveContainer. ResponsiveContainer is a special component that makes responsive any chart it’s wrapped around. In order to make your scatterplot responsive, then, you simply need to wrap it in this container!

Start by importing ResponsiveContainer into your “App.js” file along with the other chart components:

import {
  ScatterChart, Scatter, XAxis, YAxis, CartesianGrid, Tooltip, ResponsiveContainer
} from 'recharts';

As ResponsiveContainer will control the height and width of your chart, you no longer need to specify these in your <ScatterChart> element; rather, you need to specify them in the <ResponsiveContainer> element.

Delete the height and width from <ScatterChart> and, instead, add a new height of 400px (height={400}) to <ResponsiveContainer>. This sets the height of your chart to 400px, but allows the width to change based on the width of the screen. In the end, your entire render() function should look something like this:

  render() {
    const { locations, numberOfEvents, events } = this.state;
    return (
      <div className="App">
        <h1>Meet App</h1>
        <h4>Choose your nearest city</h4>
        <CitySearch updateEvents={this.updateEvents} locations={locations} />
        <NumberOfEvents
          updateEvents={this.updateEvents}
          numberOfEvents={numberOfEvents}
        />       
         <h4>Events in each city</h4>

        <ResponsiveContainer height={400} >
          <ScatterChart margin={{ top: 20, right: 20, bottom: 20, left: 20 }}>
            <CartesianGrid />
            <XAxis type="category" dataKey="city" name="city" />
            <YAxis
              allowDecimals={false}
              type="number"
              dataKey="number"
              name="number of events"
            />
            <Tooltip cursor={{ strokeDasharray: "3 3" }} />
            <Scatter data={this.getData()} fill="#8884d8" />
          </ScatterChart>
        </ResponsiveContainer>
        <EventList events={events} />
      </div>
    );
  }

Save your file and take another look at your app in the browser. What happens when you change the width of the browser window now? Does the width of the chart change in response to the change in window size?

Amazing! You now have a pretty cool looking chart that not only informs your users how many events are taking place in cities around the world, but also resizes itself automatically based on the width of the browser window—not too shabby!
